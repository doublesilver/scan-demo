사업 환경

항목 내용
사업 포지션 대한민국에서 거의 모든 워치 스트랩을 판매하는 업체
제품 재질 메탈, 실리콘, 패브릭, 가죽 등 10종 이상

현재 관리 방식 구글시트 3개 탭 (20mm 일반형 / 갤럭시 울트라 / 티렉스)
현재 규모 약 176개 제품, 호환 기종 14개+, 컬럼 29개

데이터 특성

스트랩은 재질 × 커넥터 × 호환 기종의 조합으로 제품이 정의됩니다.

같은 스트랩이라도 호환되는 기종이 다르면 별도 SKU가 생성되며, 기종이 추가될 때마다 데이터가 횡으로 확장되는 구조적 문제가 발생합니다.

현재 문제점
구글시트 기반 관리의 한계입니다.

데이터 구조 문제

  • 유사 외형 제품 많아 내부 구분 혼선
  • 제품명 / 옵션명 / SKU 기준 불일관
  • 기종 추가 시 컬럼이 무한 확장
  • 구글시트의 관계형 구조 한계

운영 문제

  • 도매 / 소매 / 공급가 / 판매가 분리 관리 어려움
  • 채널별 옵션명 규칙 별도 관리 불가
  • 콘텐츠 제작 시 데이터 수동 복사
  • 신규 기종 추가 시 전체 시트 수정 필요

구체적 사례

상황 현재 방식 문제
기종 14개 → 15개 추가 모든 제품 행에 컬럼 추가 시트 가로 확장, 관리 복잡도 증가

데이터 구조
6개 테이블로 스트랩 제품의 속성 관계를 정규화합니다.

ERD

erDiagram PRODUCT ||--o{ SKU : has PRODUCT ||--o{ ATTRIBUTE : has SKU }o--o{ COMPATIBLE_MODEL : supports SKU ||--o{ PRICE : has SKU ||--o{ CONTENT : generates

테이블 설명

테이블 역할 주요 필드
PRODUCT 제품 마스터 제품명, 재질, 커넥터, 폭(mm), 이미지
ATTRIBUTE 속성 (EAV) 속성명, 속성값 — 제품별 가변 속성 저장
SKU 판매 단위 SKU 코드, 바코드, 색상, 사이즈
COMPATIBLE_MODEL 호환 기종 기종명, 브랜드, 연결 방식 — SKU와 M:N
PRICE 가격 정보 채널(쿠팡/네이버/도매), 판매가, 공급가
CONTENT 콘텐츠 채널별 상품명, 옵션명, 상세페이지 링크
핵심: 호환 기종이 추가되어도 COMPATIBLE_MODEL에 행 하나를 추가하면 됩니다. 컬럼이 늘어나지 않습니다.

핵심 기능
자동화 기능입니다.

상품명 자동 생성

규칙: 브랜드명 + 재질 + 커넥터 + 호환 모델 + 특징

예: 스페이스쉴드 실리콘 스트랩 퀵릴리즈 갤럭시 워치 5 Pro 20mm

규칙 템플릿을 관리 화면에서 직접 수정할 수 있습니다. 수정 시 전체 상품명 일괄 재생성됩니다.

옵션명 자동 생성

채널 제약 처리 방식
네이버 옵션명 25자 이내 축약 규칙 자동 적용 + 수동 오버라이드 가능
쿠팡 옵션명 50자 이내 전체 속성 포함, 자동 생성

콘텐츠 연동 / 구글시트 동기화

콘텐츠 연동

  • 상세페이지 제작 시 DB 데이터 자동 연결
  • 속성 변경 시 연결된 콘텐츠 자동 업데이트

구글시트 동기화

  • 시트 변경 → DB 자동 반영
  • DB 변경 → 시트 역방향 동기화
  • 충돌 시 DB 우선 + 로그 기록
